# Commit and Push

현재 변경사항을 커밋하고 원격 저장소에 푸시합니다.

## Instructions

1. `git status`로 변경된 파일 확인
2. `git diff`로 변경 내용 확인
3. 변경 내용을 분석하여 적절한 커밋 메시지 작성
4. 모든 변경사항을 스테이징 (`git add .`)
5. 커밋 메시지 형식:
   - 한글로 작성
   - 첫 줄: 변경 요약 (50자 이내)
   - 빈 줄
   - 본문: 상세 변경 내용 (선택적)
6. 커밋 실행
7. 현재 브랜치를 원격 저장소에 푸시

## Commit Message Format

```
{변경 유형}: {변경 요약}

{상세 내용 (선택적)}
```

변경 유형:
- `docs`: 문서 변경
- `feat`: 새 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `chore`: 기타 변경
